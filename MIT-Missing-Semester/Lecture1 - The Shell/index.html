    Lecture1 The Shell

    概要

    这节课是 The Missing Semester 的第一节， 主要的内容是介绍了 shell 的基本命令。这节课是为了并不熟悉命令行的学习者准备的，因为尽管这门课的主题是相互独立的，但是绝大多数的工具都运行在 shell 上。

    笔记

    shell 的工作原理

    我们可以尝试运行下面两个命令

    Bash
    1
2
    date
echo "Hello World!"

    date 命令可以打印当前的时间，而 echo 命令将输入重新输出一遍。

    需要注意的是，下面的 echo命令与上面展示的并不一样

    Bash
    1
    echo Hello World

    也许打印在 shell 上的内容是一致的，但是下面这个命令事实上打印了两个东西，一个 Hello ，一个 World。

    在命令行中，一个命令后的参数是以空格分开的，因此上面的命令实际上将两个参数分别打印了出来。

    Bash
    1
2
3
    echo 'Hello World!'
echo "Hello World!"
echo Hello\ World!

    上面三个命令应当是等价的，'' 和 "" 有一些区别，这会在第二讲提到。

    shell 是如何理解 echo 命令的？

    计算机会有一些内置的系统文件，其中有一些文件存储了“环境变量”，这里存储了一些关键信息，例如你的用户名，你的主目录在哪里。其中还有一个重要的信息：路径变量（path variable）。

    可以通过下面的命令查看计算机上的路径变量：

    Bash
    1
    echo $PATH

    这个命令会返回一连串冒号分割的路径。

    当我们在 shell 中输入一个命令，比如 echo 的时候，计算机将会执行一个程序。这个程序将遍历这个列表里的所有路径，直到找到一个叫做 echo 的程序或文件，并尝试运行它。

    我们可以通过下面这个命令查看哪个程序运行了 echo：

    Bash
    1
    which echo

    shell 导航

    下面的内容会关注于如何在 shell 中导航（navigate）。

    在开始之前，如果你不知道现在你所在的工作目录，可以运行下面这个命令：

    Bash
    1
    pwd

    pwd会打印当前工作目录（print working directory）

    如果需要更改目录，可以运行下面这个命令：

    Bash
    1
    cd /home/southerwood

    cd 会更改工作目录 (change directory)

    在路径表示中，还有几个特别的用法：

    Bash
    1
2
3
4
5
    cd .    # .  进入当前目录
cd ..   # .. 进入上一层目录
cd -    # -  进入原来所在的工作目录
cd ~    # ~  进入当前用户的主目录
cd      #    cd 后留空，回到当前用户的主目录

    有时候，我们不仅需要知道当前的工作目录，还需要知道当前目录下有什么东西，这时候可以运行下面这个命令：

    Bash
    1
    ls

    ls 是一个很有用的命令，不过更经常使用的是：

    Bash
    1
    ls -l

    这个命令将会给出文件的一些更详细的信息，包括文件类型、权限、所有者、修改时间和文件大小等等。

    关于如何理解 Linux 下的权限的内容，请移步至练习题的Q6.

    在一个命令后以 - 开头的东西叫做标志（flag）或者选项（option）。需要注意的是，这两个并不是一回事，通常来说，flag 之后并不再需要一个参数，而 option 后会需要给出参数。 例如，在上面这个命令中 -l 是一个 flag，因为它不需要给出其他参数就能工作。

    如果想要了解一个命令有哪些标志和选项，可以使用 --help 来查看，例如，下面这个命令会给出关于 ls 的一些选项。

    Bash
    1
    ls --help

    一个更有用的命令可能是 man ，它接受一些命令作为参数，返回这些命令的用户手册（manual pages）.

    例如，下面这个命令会告诉你该如何使用 ls 命令：

    Bash
    1
    man ls

    shell 基本文件操作

    下面这个命令将 index.md 从当前目录移动到上一级目录，并重命名为 test.md .

    Bash
    1
    mv index.md ../test.md

    mv 命令将会移动文件（move），也可以利用它将一个文件重命名。

    下面这个命令复制一份当前目录下的 index.md，将其移动到上一级目录，并重命名为 test.md .

    Bash
    1
    cp index.md ../test.md

    cp 命令将会复制文件（copy）.

    下面这个命令将删除当前目录下的 index.md .

    Bash
    1
    rm index.md

    rm 命令将会删除文件（remove）.

    需要注意的是，默认情况下 rm 命令不能删除文件夹，如果需要这样做，可以给它一个 flag.

    Bash
    1
    rm -r test

    这个命令会删除 test 文件夹下所有内容，因为它会为文件夹下的每一个文件递归（recursion）地调用 rm .

    rm -r 事实上是一个危险的命令，你将有可能在不经意间丢掉了重要的文件，删除文件夹还有一个更安全的命令：

    Bash
    1
    rmdir test

    这个命令同样会删除文件夹，但是只允许删除空文件夹，因此更加安全。

    有删除文件夹，就有新建文件夹（make directory），下面这个命令在当前目录下新建一个名为 test 的文件夹：

    Bash
    1
    mkdir test

    shell 重定向

    在 shell 中，我们可以利用 < 和 > 完成一些重定向。假如当前文件夹下有两个文件，分别是 input.txt 和 output.txt ，我们可以用下面这个命令将前者的内容写到后者中去：

    Bash
    1
    cat < input.txt > output.txt

    这里，< input.txt 表示，我希望将 input.txt 文件的内容作为 cat 的输入，而 > output.txt 表示将其输出结果写到 output.txt 中去，如果希望将输出结果追加到 output.txt ，而不是覆盖其原来内容，可以使用 >> 代替 > .

    需要注意的是，< 重定向的目标是文件，在更多场合下，你需要的是一个更强力的重定向工具：管道符 | ，它将前一个程序的输出作为下一个程序的输入，例如下面这个命令：

    Bash
    1
    ls -l / | tail -n1

    更多实用的用法将会在后续的课程里介绍。
