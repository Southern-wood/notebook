    Lecture1 The Shell

    概要

    这节课是 The Missing Semester 的第一节， 主要的内容是介绍了 shell 的基本命令。这节课是为了并不熟悉命令行的学习者准备的，因为尽管这门课的主题是相互独立的，但是绝大多数的工具都运行在 shell 上。

    笔记

    shell 的工作原理

    我们可以尝试运行下面两个命令

    date
echo "Hello World!"

    date 命令可以打印当前的时间，而 echo 命令将输入重新输出一遍。

    需要注意的是，下面的 echo命令与上面展示的并不一样

    echo Hello World

    也许打印在 shell 上的内容是一致的，但是下面这个命令事实上打印了两个东西，一个 Hello ，一个 World。

    在命令行中，一个命令后的参数是以空格分开的，因此上面的命令实际上将两个参数分别打印了出来。

    echo 'Hello World!'
echo "Hello World!"
echo Hello\ World!

    上面三个命令应当是等价的，'' 和 "" 有一些区别，这会在第二讲提到。

    shell 是如何理解 echo 命令的？

    计算机会有一些内置的系统文件，其中有一些文件存储了“环境变量”，这里存储了一些关键信息，例如你的用户名，你的主目录在哪里。其中还有一个重要的信息：路径变量（path variable)。

    可以通过下面的命令查看计算机上的路径变量：

    echo $PATH

    这个命令会返回一连串冒号分割的路径。

    当我们在 shell 中输入一个命令，比如 echo 的时候，计算机将会执行一个程序。这个程序将遍历这个列表里的所有路径，直到找到一个叫做 echo 的程序或文件，并尝试运行它。

    我们可以通过下面这个命令查看哪个程序运行了 echo：

    which echo

    shell 导航

    下面的内容会关注于如何在 shell 中导航（navigate）。

    在开始之前，如果你不知道现在你所在的工作目录，可以运行下面这个命令：

    pwd

    pwd会打印当前工作目录（print working directory)

    如果需要更改目录，可以运行下面这个命令：

    cd /home/southerwood

    cd 会更改工作目录 (change directory)

    在路径表示中，还有几个特别的用法：

    cd .    # .  进入当前目录
cd ..   # .. 进入上一层目录
cd -    # -  进入原来所在的工作目录
cd ~    # ~  进入当前用户的主目录
cd      #    cd 后留空，回到当前用户的主目录

    有时候，我们不仅需要知道当前的工作目录，还需要知道当前目录下有什么东西，这时候可以运行下面这个命令：

    ls

    ls 是一个很有用的命令，不过更经常使用的是：

    ls -l

    这个命令将会给出文件的一些更详细的信息，包括文件类型、权限、所有者、修改时间和文件大小等等。

    关于如何理解 Linux 下的权限的内容，请移步至 Linux-权限信息

    在一个命令后以 - 开头的东西叫做标志（flag）或者选项（option）。需要注意的是，这两个并不是一回事，通常来说，flag 之后并不再需要一个参数，而 option 后会需要给出参数。 例如，在上面这个命令中 -l 是一个 flag，因为它不需要给出其他参数就能工作。

    如果想要了解一个命令有哪些标志和选项，可以使用 --help 来查看，例如，下面这个命令会给出关于 ls 的一些选项。

    ls --help

    一个更有用的命令可能是 man ，它接受一些命令作为参数，返回这些命令的用户手册（manual pages）.

    例如，下面这个命令会告诉你该如何使用 ls 命令：

    man ls

    shell 基本文件操作

    下面这个命令将 index.md 从当前目录移动到上一级目录，并重命名为 test.md .

    mv index.md ../test.md

    mv 命令将会移动文件（move），也可以利用它将一个文件重命名。

    下面这个命令复制一份当前目录下的 index.md，将其移动到上一级目录，并重命名为 test.md .

    cp index.md ../test.md

    cp 命令将会复制文件（copy）.

    下面这个命令将删除当前目录下的 index.md .

    rm index.md

    rm 命令将会删除文件（remove）.

    需要注意的是，默认情况下 rm 命令不能删除文件夹，如果需要这样做，可以给它一个 flag.

    rm -r test

    这个命令会删除 test 文件夹下所有内容，因为它会为文件夹下的每一个文件递归（recursion）地调用 rm .

    rm -r 事实上是一个危险的命令，你将有可能在不经意间丢掉了重要的文件，删除文件夹还有一个更安全的命令：

    rmdir test

    这个命令同样会删除文件夹，但是只允许删除空文件夹，因此更加安全。

    有删除文件夹，就有新建文件夹（make directory），下面这个命令在当前目录下新建一个名为 test 的文件夹：

    mkdir test

    shell 重定向

    在 shell 中，我们可以利用 < 和 > 完成一些重定向。假如当前文件夹下有两个文件，分别是 input.txt 和 output.txt ，我们可以用下面这个命令将前者的内容写到后者中去：

    cat < input.txt > output.txt

    这里，< input.txt 表示，我希望将 input.txt 文件的内容作为 cat 的输入，而 > output.txt 表示将其输出结果写到 output.txt 中去，如果希望将输出结果追加到 output.txt ，而不是覆盖其原来内容，可以使用 >> 代替 > .

    需要注意的是，< 重定向的目标是文件，在更多场合下，你需要的是一个更强力的重定向工具：管道符 | ，它将前一个程序的输出作为下一个程序的输入，例如下面这个命令：

    ls -l / | tail -n1

    更多实用的用法将会在后续的课程里介绍。

    Linux 权限信息

    文件权限

    权限信息由九个字符表示，将每三个分为一组，三组分别表示： 1. 文件所有者的权限 2. 文件所有者所在的用户组的其他用户的权限 3. 其他所有用户的权限

    每一组的权限用三个字符表示，按先后顺序分别是： 1. r -> 读取权限（read） 2. w -> 写入权限（write） 3. x -> 执行权限（execute）

    如果未拥有某个权限，该权限的位置将会被 - 代替。

    在这个例子中，rw-rw-r-- 表示，该用户的拥有者 southernwood 和其所在组的其他用户拥有对文件的读取和写入权限，但是并没有执行权限；而其他用户只拥有读取权限，这就是无法执行该程序的原因。

    目录权限

    值得注意的是，在权限信息之前还有一个字符，它表示文件的类型，在这个例子里，文件的类型是-，这表示该文件是一个 file，我们也很容易找到类型为 d 的文件，这些文件是目录（directory）。

    一个目录也有它的权限信息，但是其含义会稍微复杂一些，更难理解：

    read 权限

    允许查看该目录下的所有文件。如果希望对该文件使用 ls命令的话，你会需要这个权限。

    wirte 权限

    允许修改该目录的内容。

    可以将 ls的返回值理解为目录文件的内容，如果你拥有当前目录的 write 权限，那么你就可以在该目录下删除文件，重命名，或者新建文件。

    这里有一件有些反直觉的事情：假设当前目录下有一个 test.txt 文件，你拥有对他的写入权限，却没有当前目录的写入权限；那么你可以随意修改甚至清空该文件，但是你无法将它删除或者重命名，或者在当前目录下复制一份副本。

    execute 权限

    目录的执行权限有些让人迷惑，粗略地说，执行权限决定了你能否 cd进入这个目录，同时，如果想要访问某个目录，你需要拥有从根目录一路到这个目录的所有目录的执行权限。

    同时，对目录执行 ls -l 也需要执行权限，因为提供更详细的信息需要进入该目录。当然，也需要其所有父目录的权限。

    Q7：使用sh semester重新运行程序，并试着理解这为什么有效

    southernwood@x1-carbon:/tmp/missing$ sh semester
HTTP/1.1 200 Connection established
...

southernwood@x1-carbon:/tmp/missing$ which sh
/usr/bin/sh
southernwood@x1-carbon:/tmp/missing$ ls -l /usr/bin/sh
lrwxrwxrwx 1 root root 4 10月 19 13:38 /usr/bin/sh -> dash

    我们拥有 sh 的执行权限，以及 semester的读取权限，因此事实上我们调用了sh权限，并读取了semester中的内容，要求sh执行其中的命令。

    可以借助以下办法验证这个猜想：

    southernwood@x1-carbon:/tmp/missing$ ls -l
总用量 4
-rw-rw-r-- 1 southernwood southernwood 60 10月 21 18:46 semester
southernwood@x1-carbon:/tmp/missing$ chmod 444 semester
southernwood@x1-carbon:/tmp/missing$ ls -l
总用量 4
-r--r--r-- 1 southernwood southernwood 60 10月 21 18:46 semester
southernwood@x1-carbon:/tmp/missing$ sh semester
HTTP/1.1 200 Connection established
...

    可以看到，我们修改了semester的权限，使其只能被读取，然而sh semester可以正确运行。

    southernwood@x1-carbon:/tmp/missing$ chmod 333 semester
southernwood@x1-carbon:/tmp/missing$ ls -l
总用量 4
--wx-wx-wx 1 southernwood southernwood 60 10月 21 18:46 semester
southernwood@x1-carbon:/tmp/missing$ sh semester
sh: 0: cannot open semester: Permission denied

    而当我们修改权限为可写、可执行、不可读时，sh semester将不能运行。

    southernwood@x1-carbon:/tmp/missing$ man chmod

    Q9：使用chmod修改文件权限，让./semester可以运行，并查阅相关资料理解 shell 是如何知道应该用 sh执行semester的

    文件权限事实上使用一个三位的二进制数表示，因此，二进制下的111等同于权限rwx，而010等同于权限-w-，理解这一点之后可以很方便的设置文件权限，只需要使用一个八进制数就可以设置一个文件的权限，例如：

    southernwood@x1-carbon:/tmp/missing$ chmod 764 semester
southernwood@x1-carbon:/tmp/missing$ ls -l
总用量 4
-rwxrw-r-- 1 southernwood southernwood 60 10月 21 18:46 semester

    这里 7就是二进制下的111，对应rwx，6对应110和rw-，1对应001和r--，这与ls -l得到的结果一致。

    在修改权限后，./semester 可正确运行。

    这里有个小插曲，事实上，在修改后，./semester 在我的计算机上仍不能正确运行（我使用的操作系统是 Ubuntu 22.10 ），但这是因为 Ubuntu 将 sh 放在了 /usr/bin/sh 而不是 bin/sh，使用文本编辑器将第一行修改后，可以正确运行。

    修改前：

    southernwood@x1-carbon:/tmp/missing$ ./semester
bash: ./semester: bin/sh: 错误的解释器: 没有那个文件或目录

    修改后：

    southernwood@x1-carbon:/tmp/missing$ cat semester
#!/usr/bin/sh
curl --head --silent https://missing.csail.mit.edu
southernwood@x1-carbon:/tmp/missing$ ./semester
HTTP/1.1 200 Connection established
...

    有了这个小插曲，shell 如何找到该命令已经真相大白了。

    在脚本开头的，一行以#!开头的语句被称为 shebang，在类 Unix 系统上，程序解释器将解析#!后面的内容，并将其作为执行脚本的程序。这就是为什么修改它之前，上面的脚本在我的机器上无法执行，因为它提供了错误的程序路径。

    在许多脚本语言中，从#开始到该行结束的内容属于注释，并不会被执行，其他不以#为注释记号的语言也大多会自动忽略#!来兼容 shebang.

    Q10：使用| 和 > 将 semester 返回的文件中的 last-modified 日期输出到一个文件中，这个文件的文件名为 last-modified.txt ，并位于当前用户的主目录下

    southernwood@x1-carbon:/tmp/missing$ ./semester | grep last-modified > ~/last-modified.txt
